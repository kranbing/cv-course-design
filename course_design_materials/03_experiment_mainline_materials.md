# 实验主线素材

## 实验目标

在不进行本机训练的前提下，基于论文公开结果和官方代码结构，构建一条完整、可解释的模拟复现实验主线：

1. 说明城市道路语义分割的任务输入、输出和评价方式。
2. 以 SegFormer-B0 为核心复现模型。
3. 以传统 CNN 分割模型和 PIDNet-S 为对比对象。
4. 在 SegFormer-B0 上设计边界增强分支，模拟其对边缘类别和小目标的改善。
5. 用表格和可视化示意图展示对比、消融和误差分析。

## 数据集设定

建议报告中使用 CamVid 作为主要实验数据集，原因是：

- 场景与城市道路语义分割高度相关。
- 类别数量适中，常用于实时语义分割论文。
- PIDNet 官方 README 给出了 CamVid 公开结果，便于模拟复现对照。
- 相比 Cityscapes，CamVid 更适合作为课程设计中的轻量实验对象。

模拟设定：

- 输入尺寸：`360 x 480` 或 `512 x 512`。
- 类别数：11 类。
- 数据增强：随机缩放、随机裁剪、水平翻转、颜色扰动。
- 训练轮数设定：80 epochs。
- 优化器：AdamW。
- 初始学习率：`6e-5`。
- 损失函数：交叉熵损失 + 边界 BCE 损失。

## 方法设置

| 方法 | 类型 | 作用 |
|---|---|---|
| UNet | CNN baseline | 作为传统编码器-解码器基线 |
| DeepLabV3+ | CNN baseline | 作为空洞卷积和 ASPP 基线 |
| SegFormer-B0 | 主复现模型 | 体现轻量 Transformer 分割能力 |
| PIDNet-S | 公开结果对照 | 体现实时分割与边界分支优势 |
| Ours: SegFormer-B0 + Boundary Head | 本文改进 | 检验边界监督是否提升细节分割 |

## 本文改进方法

### 结构

在 SegFormer MLP decoder 的融合特征 `_c` 后增加一个轻量边界分支：

```text
F = MLPDecoder(C1, C2, C3, C4)
Y_sem = Conv1x1(F)
Y_edge = Conv3x3(F) -> BN -> ReLU -> Conv1x1(F)
```

语义分割输出 `Y_sem` 用于类别预测，边界输出 `Y_edge` 用于预测二值边界图。

### 边界标签生成

从语义标签 `Y` 中生成边界标签：

1. 对语义标签做形态学膨胀和腐蚀。
2. 二者差值得到类别边界区域。
3. 将边界区域转为二值 mask。

### 联合损失

总损失：

```text
L = L_ce(Y_sem, Y) + lambda * L_bce(Y_edge, B)
```

其中 `B` 是边界标签，`lambda` 为边界损失权重。模拟消融中设置 `lambda = 0, 0.2, 0.4, 0.6`。

## 模拟实验设计

### 定量对比

主要比较：

- mIoU：整体分割精度。
- Pixel Accuracy：像素分类准确率。
- Boundary F1：边界区域预测质量。
- Params：参数量。
- FPS：推理速度。

### 消融实验

1. 不加入边界分支。
2. 加入边界分支但不加边界损失。
3. 加入边界分支并使用不同 `lambda`。
4. 分析边界增强对小目标和轮廓区域的作用。

## 模拟结果解释口径

报告中应避免表述为“真实训练得到”。建议使用：

- “为验证方法趋势，本文参考公开论文结果并结合模型复杂度构建模拟复现实验。”
- “模拟结果主要用于说明不同结构设计对指标的影响趋势。”
- “由于未在本机完整训练，绝对数值不作为严格性能声明，重点分析相对变化。”

## 预期实验结论

1. UNet 和 DeepLabV3+ 能完成道路场景分割，但对远处小目标和边界细节处理不足。
2. SegFormer-B0 借助多尺度 Transformer 特征，在 mIoU 上优于传统 CNN baseline。
3. PIDNet-S 在速度上优势明显，说明实时分割需要更轻量的 CNN 分支设计。
4. 本文加入边界增强后，整体 mIoU 小幅提升，Boundary F1 提升更明显。
5. 改进代价是参数量和推理时间略有增加，但仍保持轻量。
